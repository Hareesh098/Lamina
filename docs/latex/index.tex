\hypertarget{index_autotoc_md2}{}\doxysection{\texorpdfstring{Overview}{Overview}}\label{index_autotoc_md2}
{\bfseries{Lamina}} is a modular 2D molecular dynamics (MD) simulation package designed for simulating hybrid soft solids, including spring networks and finite-\/size discs. Written in C, it models a wide variety of soft and condensed matter systems. It supports robust time evolution integrators and a range of thermostats, providing accurate force evaluations for bonded and non-\/bonded interactions.

Originally developed for 2D bonded systems, {\bfseries{Lamina}} now supports broader research goals including active matter, granular solids, and complex fluids.

\hypertarget{index_autotoc_md4}{}\doxysection{\texorpdfstring{Why "{}\+Lamina"{}?}{Why "{}\+Lamina"{}?}}\label{index_autotoc_md4}
The word {\bfseries{Lamina}} comes from Latin, meaning "{}a thin layer"{}, "{}a plate"{}, or "{}a sheet"{}. In nature and science, laminae often refer to flat, two-\/dimensional structural elements such as leaves, thin metal sheets, or tissue membranes.

This name reflects both the {\bfseries{two-\/dimensional (2D)}} nature of the simulations and the types of materials {\bfseries{Lamina}} is built to study\+: {\bfseries{liquids}}, {\bfseries{soft solids}}, and {\bfseries{networked structures}} confined to thin sheets or layers. Just as natural laminae exhibit rich structural and dynamic behaviors in simple geometry, this code explores the complexity of emergent phenomena in 2D soft matter systems.

\hypertarget{index_autotoc_md23}{}\doxysection{\texorpdfstring{Continuous Integration (CI) with Git\+Hub Actions}{Continuous Integration (CI) with Git\+Hub Actions}}\label{index_autotoc_md23}
The project includes a Git\+Hub Actions workflow ({\ttfamily .github/workflows/ci.\+yml}) that automates building and testing on Ubuntu runners with MPI installed.

The workflow performs the following\+:


\begin{DoxyItemize}
\item Checks out the latest code on push or pull requests to the {\ttfamily main} branch.
\item Installs build dependencies including {\ttfamily build-\/essential}, {\ttfamily cmake}, {\ttfamily mpich}, and MPI development libraries.
\item Configures and builds Lamina using CMake in the {\ttfamily build} directory.
\item Runs the unit tests in parallel using MPI.
\end{DoxyItemize}

You can view the build status and logs on the {\bfseries{Actions}} tab of the Git\+Hub repository.
